2020-02-25

[TOC]

应用一：分布式锁（Distributed Lock）

  • 问题来源：并发情况下，读取和修改保存这两步并不满足原子性，从而需要分布式锁来进行限制

  • 问题本质：在Redis中占个坑，防止其他进程跑进来

  • > setnx lock ex k true --保证了expire和setnx的原子性,不适用于长任务

应用二：延时队列（Delay Queue）

  • 相比rabbitMQ等专业消息队列来说实现简单方便一些

  • 直接使用Redis自带的list来实现
