Enum & Annotation

用Enum类型来替代静态域定义常量

实际上Enum的内部实现就是用static final来实现的

为什么不直接用static final int？

  • 类型不安全，不能检查到底是什么类型的，很可能会让apple和orange比较变得合法
  • 表达能力不够强大

用 实例域 替代 ordinal() 方法

ordinal方法为什么不好？

  • ordinal返回的值不固定，而是随着Enum的变化而变化

EnumSet 替代 bit域

暂时用不到

EnumMap替代ordinal 索引

不要用 ordinal建立索引，修改Enum后会错乱

用接口实现可拓展的Enum

实现某接口，将自动获取该接口内的Enum

用注解而不是命名模式（naming pattern）

naming pattern的缺点：

  • 错误命名会导致测试用例不执行
  • 不能保证程序用于指定的元素
  • 很难将参数和程序元素关联

==先学完注解再看这部分吧….难顶==

永远记得加上@override注解

标记接口定义来类型

例如：serializable、cloneable
