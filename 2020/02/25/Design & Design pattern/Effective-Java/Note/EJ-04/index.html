Generics

能加泛型的地方一定要加泛型，不要用原始的方法

泛型的优点：

  • 可以检查collections & method 被放入一些其他不需要的object
  • 安全，可读性强

消除unchecked warning

  • <>是一种消除unchecked warning的方法（靠编译器自行推算）
  • 如果能证明代码没问题，用suppressWarings（“unchecked”）来消掉

能用List不要用Array

因为Array的类型会随所存类型而变，这会让Array存在很多不确定性

优先使用generic type 而不是 raw type
