进程同步&互斥

同步

概念：进程间存在异步性，因此有时需要进程间满足一定的执行次序，同步保证了这种执行次序

互斥

概念

互斥即对资源上锁，某资源不能在同一时间多访问，因此要上锁，同Java中的synchronized

实现方式

  • 单标志法：用一个数来标志允许访问临界资源的进程PID

    • 违背了空闲让进原则

  • 双标志检查法：用一个flag[]数组来标记进程访问临界资源的意愿

    • 辣鸡方法，会导致无限等待

  • Peterson算法：同双标志法，但是主动让出进入机会

    • 还不错，但是不满足让权等待原则

硬件实现方法

  • 中断屏蔽：和原语一样，使用关中断和开中断（但是注意不适用于多处理机的情况）

  • Swap指令（XCHG）：同上，实现方式用XCHG的方式实现
